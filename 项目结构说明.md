## Android 前端文件说明

### 核心文件

| 文件 | 说明 |
|------|------|
| `MainActivity.kt` | 应用入口，管理登录状态和主屏幕 |
| `RetrofitInstance.kt` | Retrofit 单例，配置所有 API 服务 |
| `SessionManager.kt` | 基于 DataStore 的会话管理，保存认证信息 |
| `MainScreen.kt` | 主屏幕导航，管理底部导航栏和页面路由 |

### 数据层 (data/)

| 目录/文件 | 说明 |
|-----------|------|
| `api/` | 所有 API 接口定义，使用 Retrofit 注解 |
| `dto/` | 数据传输对象，与后端 API 响应对应 |
| `model/` | 本地数据模型 |

### UI 层 (ui/)

| 目录/文件 | 说明 |
|-----------|------|
| `theme/` | Material 3 主题配置，颜色、字体样式 |
| `components/` | 可复用的 UI 组件（导航栏、搜索框、卡片等）|
| `navigation/` | 页面导航和路由管理 |
| `screens/` | 各个页面的 Compose UI 实现 |

### 业务模块

| 目录 | 说明 |
|------|------|
| `community/` | 社区功能（帖子、点赞、评论）|
| `learning/` | 学习资源（视频浏览、分类、收藏）|
| `analysis/` | 动作分析（练习历史、视频上传、分析结果）|
| `profile/` | 个人中心（资料编辑、收藏管理）|

## Spring Boot 后端文件说明

### 核心配置

| 文件 | 说明 |
|------|------|
| `application.yml` | 应用配置（数据库、JWT、文件上传等）|
| `SecurityConfig.java` | Spring Security 配置，定义安全规则 |
| `FileUploadConfig.java` | 文件上传路径配置 |

### 实体类 (entity/)

每个实体对应数据库的一张表：

| 实体 | 对应表 | 说明 |
|------|--------|------|
| `User` | users | 用户信息 |
| `Post` | posts | 社区帖子 |
| `Comment` | comments | 帖子评论 |
| `VideoCategory` | video_categories | 视频分类 |
| `VideoResource` | video_resources | 学习视频资源 |
| `PracticeHistory` | practice_history | 练习历史记录 |
| `Favorite` | favorites | 用户收藏 |
| `LikeRecord` | like_records | 点赞记录 |

### 业务模块

每个模块包含 Service（业务逻辑）和 Controller（API 接口）：

| 模块 | Service | Controller | 路径前缀 |
|------|---------|------------|----------|
| 认证 | `AuthService` | `AuthController` | /api/auth |
| 社区 | `CommunityService` | `CommunityController` | /api/community |
| 学习资源 | `LearningService` | `LearningController` | /api/learning |
| 动作分析 | `AnalysisService` | `AnalysisController` | /api/analysis |
| 个人中心 | `ProfileService` | `ProfileController` | /api/profile |

## 架构说明

### Android 前端架构

采用 MVVM 架构模式：

- **Model**: 数据层（Repository、DTO）
- **View**: Compose UI 页面
- **ViewModel**: 业务逻辑和状态管理

### 后端架构

采用分层架构：

- **Controller**: API 接口层
- **Service**: 业务逻辑层
- **Repository**: 数据访问层
- **Entity**: 数据实体层

### 通信方式

- 前后端通过 RESTful API 通信
- 使用 JWT 进行身份认证
- JSON 格式数据交换

## 扩展开发

### 添加新功能模块

1. **前端**：
   - 创建对应的 ViewModel (`xxx/ XxxViewModel.kt`)
   - 创建对应的 Screen (`ui/screens/ XxxScreen.kt`)
   - 在 `MainScreen.kt` 中添加路由

2. **后端**：
   - 创建对应的 Entity (`entity/ Xxx.java`)
   - 创建对应的 Repository (`repository/ XxxRepository.java`)
   - 创建对应的 Service (`xxx/ XxxService.java`)
   - 创建对应的 Controller (`xxx/ XxxController.java`)

### 集成动作分析模型

详见 `AnalysisService.java` 中的 `analyzeVideo()` 方法，该接口已预留用于集成训练好的舞蹈分析模型。
